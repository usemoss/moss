# Contributing to MOSS

Welcome! 👋  
We’re excited that you’re interested in contributing to **MOSS** — Minimal On-Device Semantic Search.

This project is early but growing fast. Your feedback, issues, suggestions, and code contributions are very welcome!

---

## 🛠 How to Contribute

There are many ways to contribute:

- **Open an issue** for bugs, feature requests, or improvements
- **Suggest documentation updates** or examples
- **Submit a pull request** with bugfixes, tests, or new functionality
- **Share feedback** about how MOSS works for your projects

We especially appreciate contributions that help make MOSS **faster**, **smaller**, and **even easier for developers to use**.

---

## 📋 Contribution Guidelines

- Before submitting major changes, **open an issue or discussion** first so we can align on direction.
- Please keep pull requests **focused and minimal** — one feature or fix per PR.
- Follow the existing **code style and structure**.
- Add simple tests if your PR changes functionality.
- Keep documentation updated if necessary (README, examples).

---

## 🚀 Getting Started

1. Fork this repository
2. Create a new branch (`git checkout -b feature/my-new-feature`)
3. Make your changes
4. Commit your changes (`git commit -am 'Add some feature'`)
5. Push to your branch (`git push origin feature/my-new-feature`)
6. Create a new Pull Request!

---

## 📄 Code of Conduct

We’re building a welcoming and inclusive community around MOSS.

Please be respectful, constructive, and kind to others.  
We follow the [Contributor Covenant Code of Conduct](https://www.contributor-covenant.org/).

---

## 📬 Questions?

Feel free to open an issue or email us at [contact@moss.dev](mailto:contact@moss.dev).  
We'd love to hear from you!

Thanks for helping us make on-device AI search better for everyone! 🚀
